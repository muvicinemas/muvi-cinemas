Why: The F&B service handles all food & beverage ordering (concessions at the cinema). Without it, customers cannot order food with their tickets. Register Main, Identity, Payment, Notification with Internal ALB target groups

Why: Without ALB target groups, these services rely only on Cloud Map DNS for discovery. If a container crashes, DNS cache means traffic continues routing to the dead container for 60-120 seconds. ALB health checks detect failures in 10 seconds and stop routing immediately. This directly affects payment reliability ‚Äî a 2-minute window of failed payments is unacceptable.

Container count: UAE runs 2 containers per service vs Frankfurt's 1

Why it's fine: More containers = better availability during deployments (rolling updates need at least 2 to maintain uptime). Your team lead actually improved on the vendor's setup here. This costs a bit more but ensures zero-downtime deploys.

Create Internal ALB (Muvi-Internal-UAT)

Why: The internal ALB is how microservices communicate with each other securely inside the VPC. The gateway forwards requests to identity/main/payment/notification through this ALB. Without it, inter-service communication has no load balancing and no health checking. When gRPC calls from gateway ‚Üí payment fail because a container is unhealthy, customers get payment errors.

KEEP AS-IS

Separate Website ALB is not required

Why it's fine: Frankfurt has a dedicated ALB for the website, but you can use host-based routing rules on the single public ALB to route website traffic. One ALB can serve multiple services via different hostnames. Saves ~$18/month.

Why: Without proper domains, the mobile app and website cannot point to the UAE environment. The app is hardcoded to call api.uat.muvicinemas.com ‚Äî if that still routes to Frankfurt's CloudFront, you're not actually testing the UAE setup. Why: A UAT environment exposed to the internet without WAF is an open attack vector. Bots, vulnerability scanners, and attackers will find it. WAF blocks SQL injection, XSS, and rate-limits abusive requests. Basic WAF with AWS Managed Rules costs ~$6/month ‚Äî negligible cost, significant security. If UAT gets compromised, attackers get database access to real customer test data.

Why: Without pipelines, every deployment requires someone to manually build a Docker image, push it to ECR, and update the ECS task definition through the AWS Console. This is:

Recommendation: Use GitHub Actions (team already uses GitHub). Workflow files live in the repo, are version-controlled, and the team is familiar with them. Each service gets a .github/workflows/deploy-uat-uae.yml.
