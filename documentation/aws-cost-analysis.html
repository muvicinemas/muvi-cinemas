üí∞ Cost-Saving Tricks the Vendor Uses (for UAT)

üåô Scale-to-Zero at Night
Saves ~30-40% on ECS costs
Min capacity = 0 on gateway, website, notification, identity. Services can shut down completely when nobody's testing at night.
üóÑÔ∏è Single-AZ Databases
Saves ~50% on RDS costs (~$533 saved)
No backup copy of databases in a second data center. If the server dies, there's downtime. Fine for testing, risky for production.
üí® Burstable Instances (t3 family)
Saves ~60% vs dedicated (m5/r5)
t3 instances share CPU with other customers. They can "burst" when needed but throttle under sustained load. Perfect for unpredictable UAT traffic.
üõë Stopped EC2 Instances
Saves ~$45/month
4 of 5 EC2 jump boxes are stopped. You don't pay for stopped compute ‚Äî only their storage volumes (~$2/month each).
‚è∞ RDS Start/Stop Schedule (DISABLED)
Could save ~$15/day when active
They had Lambda functions to stop all databases at 9PM and start at 5AM. Currently disabled ‚Äî probably someone got tired of databases being down during late-night testing.
üì¶ Small Container Sizes
Minimal Fargate spend
Most services run on 0.5 vCPU / 1 GB ‚Äî the smallest practical Fargate configuration. Only F&B uses more (2 vCPU / 4 GB).
